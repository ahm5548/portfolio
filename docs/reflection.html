Comparing GitHub Pages and My Other Development Method

While creating my websites, I worked with GitHub Pages and another website development environment. The most significant difference between the two methods is how they handle publishing and editing. GitHub Pages requires uploading files to a repository and managing versions, while the other platform allows more direct editing and immediate visual feedback.

Challenges for a New Learner

One of the biggest challenges when using GitHub Pages is understanding how repositories, commits, and file structures work. I found it extremely hard to commit to memmory all the commands needed in the shell. It can be confusing at first to keep track of where files are located and how to correctly link them. In the other development environment, the challenge is learning the interface and tools, especially if it includes built-in editors or design features that behave differently from writing raw code.

Advantages of Each Method

GitHub Pages offers strong advantages in version control and organization. It makes it easy to track changes and maintain a professional workflow. The other development method is more beginner-friendly, as it often provides a visual interface and faster feedback when making changes, which helps speed up the design process.

Challenges in Layout and Organization

Modifying layout in GitHub Pages requires editing HTML and CSS directly, which can be difficult without a strong understanding of coding. However, it also gives full control over the design. In the other platform, layout changes may be easier using built-in tools, but sometimes those tools can be limiting or less flexible compared to coding everything manually.

Skills Needed for Development

Both methods require knowledge of HTML and CSS. GitHub Pages also requires understanding file management, version control, and basic Git commands. I found my other website harder to make as it didnt allow for as much individual editing and relied heavily on themes for their pages.

Skills I Want to Improve

In improving my websites I realized that I need to spend more time going over css code and learning how it works, I think this could be helpful because learning all the different styling code greatly affects how your website looks. I also want to improve my understanding of changing my design so my sites look good on all devices. Additionally, learning more about xml and making websites using themes. would help me add interactivity to my pages.
